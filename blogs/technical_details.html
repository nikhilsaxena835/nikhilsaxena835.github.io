Technical Architecture

Behind the scenes of World Album Discovery

How It Works

To achieve an unbiased and high-quality collection of albums, the platform relies on a sophisticated data pipeline and a serverless edge architecture.

1. Initial Artist Seeding (getArtists.py)

The curation begins with geography. Using a Python script (getArtists.py), the platform executes highly optimized SPARQL queries against Wikidata. It fetches the top 50 musical artists and groups from over 20 different countries. The script strictly filters for entities classified as musicians, singers, composers, or musical groups, sorting them by their Wikipedia site link count to gauge historical or cultural relevance.

2. Data Enrichment and Validation (seed.js)

Once the raw list of artists is gathered, a Node.js pipeline (seed.js) takes over to validate and enrich the data. This pipeline interacts with two major external APIs:

  • Last.fm API: Used to strictly validate the artist's global listener count. High thresholds (e.g., a minimum of 30,000 listeners) are enforced to systematically filter out non-musicians, politicians, or vanity projects that might have slipped through Wikidata.
  • MusicBrainz API: For the validated artists, the pipeline queries MusicBrainz to fetch their official "Studio Albums" (explicitly rejecting live albums, compilations, and soundtracks). It then fetches the corresponding cover art via the Cover Art Archive.

3. Serverless Infrastructure

The entire application is hosted on Cloudflare Pages using a serverless architecture, which means there is no traditional backend server running continuously.

  • Cloudflare D1 (SQLite): The curated albums, artists, genres, and countries are stored in D1, Cloudflare's serverless SQLite database. This database also stores user profiles and their interaction history (Liked, Disliked, Skipped albums) through foreign-key enforced relationships.
  • Cloudflare Functions: API routes (like /api/discover or /api/interact) run entirely at the edge. They query the D1 database directly and serve the frontend dynamically.
  • Cloudflare KV CACHE: Used for fast, globally distributed key-value storage at the edge, ensuring the application remains blazing fast regardless of where the user is located in the world.
